🎯 实现的功能（11 个 API 接口）
================================================================================

系统接口（2 个）
├── GET /health                      ✅ 健康检查
└── GET /                            ✅ API 入口

大模型接口（3 个）
├── POST /process/check_llm_connection   ✅ 测试大模型连通性
├── POST /process/generate              ✅ 大模型生成 CSV 数据
└── POST /process/ai_chat               ✅ 大模型对话

数据处理接口（3 个）
├── POST /process/clean              ✅ 数据清洗
├── POST /process/features           ✅ 特征工程（独热编码）
└── POST /process/split              ✅ 数据划分（训练/测试集）

模型接口（3 个）
├── POST /model/train_manual         ✅ 模型训练
│   • 支持：随机森林、KNN、逻辑回归
│   • 返回：模型路径、精度、特征列名
├── POST /model/evaluate             ✅ 模型评估
│   • 输入：模型 + 测试集
│   • 返回：准确率、样本预测
└── POST /model/predict_manual       ⭐ 模型预测（关键特性）
    • 自动对齐特征列
    • 解决列不匹配导致的 500 错误

聚类接口（1 个）
└── POST /clustering/train_and_visualize  ✅ K-Means 聚类
    • 自动 StandardScaler 标准化
    • 自动 PCA 降维（特征 > 2D 时）
    • 返回 base64 编码的可视化图片

⭐ 核心亮点特性
================================================================================

1️⃣ 特征列自动对齐（关键特性）
   ├─ 问题：模型训练时有 10 列，预测时只有 8 列 → 500 错误
   ├─ 解决：使用 df.reindex(columns=train_cols, fill_value=0)
   └─ 结果：自动对齐、新列填 0、缺失列删除 ✅

   ├─ 保存：模型本身 + 特征列名 + LabelEncoder
   ├─ 预测时读取 train_cols，对齐输入特征

📋 API 使用示例（完整流程）
================================================================================

# 生成数据
POST /process/generate
{
  "prompt": "生成 100 行电商数据",
  "row_count": 100
}
→ 返回：file_path = "temp/data_xxx.csv"

# 清洗数据
POST /process/clean
{
  "file_path": "temp/data_xxx.csv"
}
→ 返回：cleaned_file_path = "temp/data_cleaned_xxx.csv"

# 特征工程
POST /process/features
{
  "file_path": "temp/data_cleaned_xxx.csv",
  "categorical_features": ["category"]
}
→ 返回：feature_file_path = "temp/data_features_xxx.csv"

# 数据划分
POST /process/split
{
  "file_path": "temp/data_features_xxx.csv",
  "test_size": 0.2
}
→ 返回：train_file_path, test_file_path

# 模型训练
POST /model/train_manual
{
  "file_path": "temp/train_xxx.csv",
  "target_column": "price",
  "model_type": "random_forest"
}
→ 返回：model_path, accuracy, train_cols

# 模型预测（自动对齐列）
POST /model/predict_manual
{
  "model_path": "models/model_random_forest_xxx.joblib",
  "data_file_path": "temp/test_xxx.csv"
}
→ 返回：prediction_file_path（包含预测结果）

❓ 常见问题
================================================================================

Q: 没有 API Key 能使用吗？
A: 可以！大模型相关功能会被禁用，但数据处理和模型训练仍正常工作。

Q: 模型预测时列不匹配怎么办？
A: 已自动处理！代码会自动对齐特征列到训练时的列。

Q: PCA 降维何时触发？
A: 当特征数 > 2 时，K-Means 聚类会自动使用 PCA 降维到 2D 用于可视化。

Q: 如何修改服务端口？
A: 在 main.py 最后修改 uvicorn.run 中的 port 参数，或添加命令行参数。

Q: 大模型功能不可用？
A: 检查 .env 文件中是否配置了正确的 VOLC_API_KEY。

Q: 如何在 Docker 中运行？
A: 运行 `docker-compose up -d`，或 `docker run` 配合参数。

Q: 如何扩展新的模型类型？
A: 在 train_manual 函数中添加新的 elif 分支，参考现有的模型配置。

Q: 如何添加新的 API 接口？
A: 在 main.py 中添加新的 @app.post 或 @app.get 装饰器的函数即可。
