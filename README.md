# AImake 项目说明

## 1. 这个项目能干什么
AImake 是一个“Java + Python + 原生 HTML” 的机器学习教学与实操平台，目标是让用户在网页里完成从数据到模型结果的完整流程。

核心能力（按用户视角）：
- 上传 Excel/CSV 数据并预览（支持前端分页展示）
- 通过 AI 生成数据（可用于快速演示或无数据场景）
- 执行数据清洗、异常值检测、特征处理等预处理步骤
- 支持有监督学习流程（训练、评估、手动输入预测）
- 支持无监督学习流程（聚类训练与可视化）
- 提供 AI 对话助手，用于解释步骤、辅助分析和交互问答

核心架构（按技术视角）：
- Java（Spring Boot）负责：静态页面托管、文件上传预览、统一 API 代理转发
- Python（FastAPI）负责：数据处理、模型训练与推理、算法结果输出
- 前端（HTML/CSS/JS）负责：分步引导、状态提示、交互展示

## 2. 这个项目有什么用
这个项目适合“教学演示 + 轻量实战”场景，尤其是希望快速体验机器学习全流程的人群。

典型使用场景：
- 教学网站：向学生展示数据处理到建模评估的完整链路
- 快速验证：不搭复杂数据库，直接用文件驱动完成实验
- 课程项目：前后端+算法解耦，便于分工协作
- 内部演示：非技术同学也能通过页面快速理解模型流程

## 3. 当前方案的优点
- 架构解耦清晰：Java 管“入口和转发”，Python 管“算法”，职责边界明确
- 上手门槛低：不依赖数据库，核心数据通过文件路径传递，部署和联调简单
- 前端联调效率高：页面直接由 Spring Boot 提供，启动后即可一站式验证
- 接口扩展方便：Java 侧有通用代理逻辑，新增 Python 接口时改动较小
- 调试可定位：关键流程有日志和 requestId，便于追踪问题
- 教学友好：分步式流程 + 提示信息 + 可视化结果，符合教育场景

## 4. 可优化点（按优先级）

### P0（优先处理，直接影响稳定性）
- 临时文件治理：`temp/` 文件会持续增长，需要定时清理策略（按天数/大小/访问时间）
- 错误返回统一化：Java 代理层与 Python 返回格式建议统一（如 `status/message/details/requestId`）
- 接口契约收敛：减少 422/500 的关键在于前后端字段命名、类型、必填规则严格一致
- 关键链路监控：为上传、转发、训练步骤增加更结构化日志（含接口名、耗时、请求ID）

### P1（建议尽快做，提升体验和可维护性）
- API 文档标准化：补全接口入参/出参样例与错误码，降低前后端沟通成本
- 代理转发增强：透传更多请求头、响应头和状态细节，减少“黑盒转发”问题
- 前端状态机化：分步按钮可用状态由“步骤完成标记”驱动，避免流程越界
- 配置集中化：Python 基础地址、超时、调试开关统一放配置文件，减少硬编码

### P2（中长期优化，提升性能与工程质量）
- 大文件处理优化：可引入异步任务和进度查询，避免长请求阻塞
- 结果缓存与复用：相同参数下复用中间结果，减少重复计算
- 自动化测试：补齐 Java 控制器测试 + Python 接口测试 + 前端关键流程回归
- 安全与隔离：为生产环境增加上传文件类型白名单、大小限制、目录隔离策略

## 5. 范围与边界说明
- 当前定位是教学与实验型平台，不是面向高并发生产训练平台
- 项目强调“快速联调与可视化体验”，在复杂任务调度、权限体系、多租户方面仍可继续扩展

## 6. 一句话总结
AImake 的价值在于：用最小系统复杂度，把机器学习从“代码流程”变成“可视化、可教学、可交互”的完整体验；下一阶段重点是稳态治理（临时文件、契约一致性、日志与测试）。
