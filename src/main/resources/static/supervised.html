流程导航
1. 数据准备

先上传 Excel，或者让 AI 生成训练数据。

知识点：高质量样本是模型效果的基础，特征字段建议语义清晰、缺失值尽量少。
教程：这一步在做什么
  • 训练数据里的每一行都代表一个案例，包含特征和目标标签。
  • 目标标签是有监督学习的关键，没有标签就无法训练分类或回归模型。
  • 建议优先使用业务真实数据，再用 AI 生成样例补充边界场景。
2. 数据预处理

先一键清洗，再按需做特征工程。

知识点：预处理是把脏数据变成可学习数据，缺失值处理和特征工程会直接影响泛化能力。
教程：为什么要预处理
  • 原始数据通常有缺失、异常和格式不统一问题，不处理会让模型学到噪声。
  • 一键清洗会先移除不完整样本，再过滤明显异常值，提高数据稳定性。
  • 特征工程会把原始字段转换成模型更容易利用的特征表示。

特征工程
3. 训练测试集划分

先把数据分成训练集和测试集，后续评估才能客观。

知识点：训练集用于学习，测试集用于验证泛化能力，常见划分比例是 8:2 或 7:3。
教程：划分背后的原理
  • 训练集用于学习规律，测试集用于检查模型是否能泛化到新样本。
  • 如果只看训练集准确率，容易出现过拟合，课堂演示会产生误导。
  • 随机种子用于复现结果，方便教学对比不同模型的表现。
4. 模型训练

填写目标列，选择模型后开始训练。

知识点：随机森林抗过拟合能力强，KNN直观但对尺度敏感，逻辑回归可解释性好。
教程：模型怎么选
  • 随机森林适合作为默认起点，通常在表格数据上有较稳表现。
  • KNN依赖距离计算，对数据尺度敏感，适合做算法原理演示。
  • 逻辑回归可解释性较好，适合讲解特征对预测结果的影响方向。

5. 检测与预测

先做测试集检测，再做手动输入预测。

知识点：检测用于验证模型稳定性，手动输入预测用于模拟真实业务场景。
教程：如何解读结果
  • 先看测试集检测指标，确认模型在未见过数据上的表现是否可接受。
  • 再做手动输入预测，模拟真实系统收到一条新数据后的推理流程。
  • 若结果异常，优先检查输入 JSON 键名是否和训练阶段特征完全一致。
